XHTML Editor usage guide

Introduction

The XHTML Editor is a simple editor based on SHEF that can be used to edit a subset of XHTML 1.0 Strict documents. It permits to edit both the source code and visually the document, as well as exporting the documents to different formats.

XHTML Support

Introduction

It currently officially supports editing the following XHTML Modules, as defined in XHTML 1.1 Modularization:

  • Structure module
  • Text Module
  • Hypertext Module
  • List Module
  • Presentation Module
  • List Module
  • Presentation Module
  • Basic Tables Module
  • Image Module
  • Style Sheet Module
  • Style Attribute Module
  • Metainformation Module

Specific module notes

The Image Module has some specificities, it currently contains the following features and limitations:

  • The editor only supports adding PNG images (MIME Type "image/png") and SVG vector images (MIME Type "image/svg+xml) for better portability
  • Embedded images in the document is supported following the DATA URI scheme, as defined in IETF RFC 2397. For portability reasons, it limits the size of the embedded data to base64 64 KiB.
  • Local images in the same location as the document are supported only once the document is first saved or when the document is opened, as the Base URL is created only when the current directory of the document is known.The actual filename of the image needs to be set in URL field of the Image Properties dialog box.The title, if found embedded in the PNG text chunk is automatically added to the title and alt attributes.

Exporting

Such as in other minimalistic typesetting systems, the document must be associated with a document class, which indicates how it will be exported. The document class is specified when exporting the document. The following document classes and their attributes are currently supported:

Document class Table of contents Title page Heading numbering
article NO NO NO
report YES YES YES
book YES YES NO
letter NO NO YES

Document properties

The following document properties are supported and will be mapped accordingly to their HTML equivalence

  • title as defined in the title element.
  • creator (author) representing the creator / authors of the document.
  • description (description), a free form string that describes the document.
  • subject (keywords) as a set of comma separated list of tokens representing keywords associated with the page.
